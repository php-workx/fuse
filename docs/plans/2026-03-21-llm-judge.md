# Plan: LLM Judge — AI-Assisted Classification Review

## Context

Fuse's rule-based classification is fast and deterministic but context-blind. Pattern matching
can't distinguish `rm -rf ./build/` (routine cleanup) from `rm -rf ~/` (catastrophic).
This leads to false positives (safe commands classified as APPROVAL/CAUTION) and false negatives
(dangerous commands slipping through as CAUTION when they should require APPROVAL).

An LLM judge provides a second opinion on CAUTION and APPROVAL classifications. It understands
command semantics, working directory context, and intent — bridging the gap between dumb pattern
matching and slow human review. The judge is optional, runs in shadow mode first to build
confidence, and uses locally-installed CLI tools (claude, codex) to avoid API key management.

**Trigger decisions:** CAUTION and APPROVAL only.
- **SAFE** — no value in judging (already cleared by rules).
- **CAUTION** — judge can downgrade to SAFE (reduce noise) or upgrade to APPROVAL (catch risks).
- **APPROVAL** — judge can downgrade to SAFE/CAUTION (reduce false positives) or confirm.
- **BLOCKED** — never overridden. Hardcoded self-protection is inviolable. If a user's policy
  blocks too broadly, they should fix their policy, not rely on an LLM to second-guess it.

Applied findings: none (new capability, no prior incidents)

## Boundaries

**Always:**
- Judge is opt-in only. Fuse works identically without it configured.
- Shadow mode is the default when enabled. Active mode requires explicit `mode: active`.
- Judge can never override BLOCKED decisions (hardcoded self-protection is inviolable).
- Judge errors and timeouts fall through silently — original classification stands.
- The LLM prompt includes the command, working directory (cwd), workspace name, current
  decision, rule ID, reason, tool name, and optionally script contents (when file
  inspection triggered the classification). Script contents are scrubbed via
  `ScrubCredentials` before inclusion. Session IDs and approval records are NOT sent.
- Prompts are passed to CLI providers via stdin pipe, never as shell arguments (prevents
  breakage from quotes, newlines, and metacharacters in commands).
- Judge response must be one of SAFE/CAUTION/APPROVAL — if the LLM returns BLOCKED or any
  unknown value, the response is rejected and the original classification stands.

**Ask First:**
- Whether to support direct API calls (Anthropic/OpenAI SDKs) in v1 or defer.
- Whether the judge should see previous decisions for the same command (approval history).

**Never:**
- Require an LLM provider for fuse to function.
- Send approval records, session tokens, or HMAC secrets to the LLM.
- Send unscrubbed script contents (always apply `ScrubCredentials` first).
- Allow the judge to override BLOCKED → anything.
- Pass prompt content as CLI arguments (use stdin pipe).
- Truncate script contents silently — if over 50KB, mark as TRUNCATED so the
  judge knows it cannot see the full file and should keep the original classification.

### 2. Provider Abstraction (`internal/judge/provider.go`)

**Interface:**

```go
type Provider interface {
    // Query sends a prompt via stdin and returns the raw response text.
    Query(ctx context.Context, systemPrompt, userPrompt string) (string, error)
    // Name returns the provider name for logging.
    Name() string
}
```

**All providers pass prompts via stdin pipe** (not CLI arguments) to avoid shell
escaping issues with quotes, newlines, and metacharacters in commands.

**Claude CLI provider:**
```go
type claudeProvider struct {
    model string // e.g., "claude-haiku-4-5-20251001"
}

func (p *claudeProvider) Query(ctx context.Context, systemPrompt, userPrompt string) (string, error) {
    cmd := exec.CommandContext(ctx, "claude",
        "--print", // read prompt from stdin, print response to stdout
        "--system-prompt", systemPrompt,
        "--model", p.model,
        "--output-format", "text",
        "--max-tokens", "256",
    )
    cmd.Stdin = strings.NewReader(userPrompt)
    out, err := cmd.Output()
    return strings.TrimSpace(string(out)), err
}
```

**Codex CLI provider:**
```go
type codexProvider struct {
    model string // e.g., "gpt-5.4-mini"
}

func (p *codexProvider) Query(ctx context.Context, systemPrompt, userPrompt string) (string, error) {
    // Codex CLI flags need verification against actual docs.
    cmd := exec.CommandContext(ctx, "codex",
        "--quiet",
        "--model", p.model,
    )
    cmd.Stdin = strings.NewReader(systemPrompt + "\n\n" + userPrompt)
    out, err := cmd.Output()
    return strings.TrimSpace(string(out)), err
}
```

**Note:** Codex CLI flags (`--quiet`, `--model`) need verification against actual
CLI documentation before implementation. The interface is stable; the invocation
details may change.

**Auto-detection:**
```go
func DetectProvider(preferred, model string) (Provider, error) {
    if preferred != "" && preferred != "auto" {
        return newProvider(preferred, model)
    }
    for _, name := range []string{"claude", "codex"} {
        if _, err := exec.LookPath(name); err == nil {
            return newProvider(name, model)
        }
    }
    return nil, fmt.Errorf("no LLM provider found (install claude or codex CLI)")
}
```

**Note on auto-detection order:** Claude CLI is tried first because it's the most
common in fuse's target environment (Claude Code users). Users who prefer a different
provider should set `provider: codex` explicitly to avoid unexpected cloud costs.

### 3. Judge Prompt (`internal/judge/prompt.go`)

**System prompt — narrow, structured, security-focused:**

```go
const systemPrompt = `You are a command safety judge for a software development environment.
Your job: evaluate whether a shell command's current safety classification is correct.

Classifications (ordered by severity):
- SAFE: routine development command, no risk of data loss or security exposure
- CAUTION: worth logging but safe to auto-approve (e.g., git push to feature branch)
- APPROVAL: requires explicit human approval — could cause data loss, expose secrets,
  affect production, or modify critical infrastructure

Important:
- You may ONLY return SAFE, CAUTION, or APPROVAL. Never return BLOCKED or any other value.
- When uncertain, keep the current classification (do not guess).
- Consider the working directory — commands in /tmp are less risky than in /.
- Consider the target — "git push --force origin main" is riskier than "git push --force origin feat/x".
- Read-only commands are almost always SAFE.
- Commands targeting production, secrets, or irreversible state changes need APPROVAL.
- Never classify as SAFE if the command could delete data outside the working directory.
- If script contents are provided, read them fully before deciding. Do not assess risk
  from just the first few lines — dangerous operations can appear anywhere in a script.
- If script contents are marked as TRUNCATED, you cannot fully assess the script.
  Keep the current classification when you cannot see the full script.

Respond with ONLY this JSON (no markdown, no explanation):
{"decision":"SAFE|CAUTION|APPROVAL","confidence":0.0-1.0,"reasoning":"one sentence"}`
```

**User prompt construction:**

The prompt includes all context needed for an informed decision. Script contents are
included when the classification was triggered by file inspection (e.g., `bash script.sh`
where the classifier couldn't statically verify the script). This lets the judge
actually read the script and assess risk, dramatically reducing false-positive APPROVALs.

```go
// maxScriptBytes is the maximum script content to send to the judge.
// Above this, the judge sees a truncation notice and should keep the original
// classification. 50KB covers virtually all real scripts.
const maxScriptBytes = 50 * 1024

type PromptContext struct {
    Command           string // the shell command or MCP tool call
    Cwd               string // working directory
    WorkspaceRoot     string // project root (last 2 path components)
    CurrentDecision   string // CAUTION or APPROVAL
    Reason            string // why this classification was assigned
    RuleID            string // which rule triggered
    ToolName          string // "Bash", "mcp__server__delete_items", etc.
    ScriptContents    string // full script contents if file inspection triggered (scrubbed)
    ScriptPath        string // path to the inspected script
}

func BuildUserPrompt(ctx PromptContext) string {
    var b strings.Builder
    fmt.Fprintf(&b, "Command: %s\n", ctx.Command)
    fmt.Fprintf(&b, "Working directory: %s\n", ctx.Cwd)
    if ctx.WorkspaceRoot != "" {
        fmt.Fprintf(&b, "Workspace: %s\n", ctx.WorkspaceRoot)
    }
    fmt.Fprintf(&b, "Current classification: %s\n", ctx.CurrentDecision)
    fmt.Fprintf(&b, "Rule: %s\n", ctx.RuleID)
    fmt.Fprintf(&b, "Reason: %s\n", ctx.Reason)
    if ctx.ToolName != "" && ctx.ToolName != "Bash" {
        fmt.Fprintf(&b, "Tool: %s\n", ctx.ToolName)
    }
    if ctx.ScriptContents != "" {
        // Scrub credentials before sending to LLM.
        scrubbed := db.ScrubCredentials(ctx.ScriptContents)
        if len(scrubbed) > maxScriptBytes {
            fmt.Fprintf(&b, "\nScript contents (%s, TRUNCATED at 50KB — cannot fully assess):\n%s\n",
                ctx.ScriptPath, scrubbed[:maxScriptBytes])
        } else {
            fmt.Fprintf(&b, "\nScript contents (%s):\n%s\n", ctx.ScriptPath, scrubbed)
        }
    }
    return b.String()
}
```

**Script content sourcing:** The `inspect` package already reads script files for
classification. When `ClassifyResult.FileInspected` is true, the adapter can read
the script file and pass it to the judge via `PromptContext.ScriptContents`. The
content is scrubbed via `db.ScrubCredentials` before inclusion in the prompt.

### 4. Judge Core Logic (`internal/judge/judge.go`)

```go
type Judge struct {
    provider           Provider
    mode               string   // "shadow", "active"
    upgradeThreshold   float64
    downgradeThreshold float64
    triggerDecisions   map[core.Decision]bool
    timeout            time.Duration
    rateLimiter        *rateLimiter // token bucket, nil if no limit
}

type Verdict struct {
    OriginalDecision core.Decision
    JudgeDecision    core.Decision
    Confidence       float64
    Reasoning        string
    Applied          bool    // true if the judge's decision was used
    ProviderName     string
    LatencyMs        int64
    Error            string  // non-empty if judge failed
}
```

**Provider caching:** `NewJudge` detects the provider once and stores it.
`MaybeJudge` caches the Judge instance via `sync.Once` so long-running adapters
(codex-shell, mcp-proxy) don't re-detect on every call. Hook mode creates a
new process per invocation anyway.

**Rate limiting:** Simple token bucket — `max_calls_per_minute` tokens, refilled
per minute. When exhausted, `Evaluate` returns nil (skip) with a slog.Warn.
Prevents runaway costs with fast agents.

**Core evaluation method:**

```go
func (j *Judge) Evaluate(ctx context.Context, result *core.ClassifyResult, promptCtx PromptContext) *Verdict {
    verdict := &Verdict{
        OriginalDecision: result.Decision,
        ProviderName:     j.provider.Name(),
    }

    // 1. Only trigger on configured decisions (CAUTION, APPROVAL)
    if !j.triggerDecisions[result.Decision] {
        return nil
    }

    // 2. Never override BLOCKED (redundant with trigger check, but defense-in-depth)
    if result.Decision == core.DecisionBlocked {
        return nil
    }

    // 3. Check rate limit
    if j.rateLimiter != nil && !j.rateLimiter.Allow() {
        slog.Warn("llm judge rate limited, skipping")
        return nil
    }

    // 4. Query the LLM with timeout
    judgeCtx, cancel := context.WithTimeout(ctx, j.timeout)
    defer cancel()

    start := time.Now()
    userPrompt := BuildUserPrompt(promptCtx) // PromptContext populated by caller
    response, err := j.provider.Query(judgeCtx, systemPrompt, userPrompt)
    verdict.LatencyMs = time.Since(start).Milliseconds()

    if err != nil {
        verdict.Error = err.Error()
        return verdict // fail-open: original classification stands
    }

    // 5. Parse and validate response
    parsed, err := ParseResponse(response)
    if err != nil {
        verdict.Error = fmt.Sprintf("parse: %v", err)
        return verdict
    }

    verdict.JudgeDecision = core.Decision(parsed.Decision)
    verdict.Confidence = parsed.Confidence
    verdict.Reasoning = parsed.Reasoning

    // 6. Apply confidence thresholds (active mode only)
    if j.mode != "active" {
        verdict.Applied = false
        return verdict // shadow mode: log only
    }

    judgeSeverity := core.DecisionSeverity(verdict.JudgeDecision)
    origSeverity := core.DecisionSeverity(result.Decision)

    switch {
    case judgeSeverity > origSeverity:
        // Upgrade (more restrictive) — lower threshold needed
        verdict.Applied = parsed.Confidence >= j.upgradeThreshold
    case judgeSeverity < origSeverity:
        // Downgrade (more permissive) — higher threshold needed
        verdict.Applied = parsed.Confidence >= j.downgradeThreshold
    default:
        // Same decision — judge agrees, no change needed
        verdict.Applied = false
    }

    return verdict
}
```

**Integration helper for adapters:**

```go
var (
    judgeOnce     sync.Once
    cachedJudge   *Judge
    cachedJudgeOK bool
)

// MaybeJudge runs the LLM judge if configured. Returns the (possibly modified)
// ClassifyResult and a Verdict for logging. If judge is disabled or not triggered,
// returns the original result and nil verdict.
func MaybeJudge(ctx context.Context, cfg *config.Config, result *core.ClassifyResult, promptCtx PromptContext) (*core.ClassifyResult, *Verdict) {
    if cfg == nil || cfg.LLMJudge.Mode == "" || cfg.LLMJudge.Mode == "off" {
        return result, nil
    }

    // Cache the judge instance (provider detection is expensive).
    judgeOnce.Do(func() {
        j, err := NewJudge(cfg.LLMJudge)
        if err != nil {
            slog.Warn("llm judge init failed", "error", err)
            return
        }
        cachedJudge = j
        cachedJudgeOK = true
    })
    if !cachedJudgeOK {
        return result, nil
    }

    verdict := cachedJudge.Evaluate(ctx, result, promptCtx)
    if verdict == nil {
        return result, nil
    }

    // Apply verdict if in active mode and confidence met
    if verdict.Applied {
        result = result.WithDecision(verdict.JudgeDecision,
            fmt.Sprintf("LLM judge (%s): %s", verdict.ProviderName, verdict.Reasoning))
    }

    return result, verdict
}
```

func (r *ClassifyResult) WithDecision(d Decision, reason string) *ClassifyResult {

- Fine-tuned judge prompt per domain (AWS, k8s, git, etc.)

- Judge training mode (user corrects judge, prompt is refined)
- Cost tracking per provider
